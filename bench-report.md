# GLM + NVIDIA 模型综合评测报告

> 测试日期：2026-05-08 ~ 2026-05-09 | 6维度评分 | 16个模型 | 质量75% + 速度25%

---

## 一、评测概述

### 评测目标
对智谱GLM系列（8个）和NVIDIA免费API模型（8个）进行中文能力综合评估，为OpenClaw智能体系统选择最优模型配置。

### 评测维度（各0-10分，满分60分）

| 维度 | 测试方法 | 评分重点 |
|------|----------|----------|
| **中文写作** | 300字介绍量子计算基本原理 | 字数(250+/150+/80+)、三个关键词覆盖、原理表述 |
| **逻辑推理** | 三开关三灯问题 | 触摸热度法、等待策略、开关编号、开/关操作 |
| **代码生成** | Python二叉树层序遍历BFS | def定义、队列使用、BFS/层序关键词、循环逻辑 |
| **数学** | 因式分解 x²+5x+6=0 | 正确解(-2,-3)、因式分解格式、验证步骤 |
| **指令遵循** | 列出5种语言(严格格式约束) | 恰好5项、无编号、字母排序、无多余文字 |
| **幻觉检测** | 1真1假1真三问 | Python/Guido正确、虚构人物拒绝回答、诺贝尔奖正确 |

### 综合评分公式
```
综合分 = 质量(总分/60×10) × 75% + 速度分 × 25%
速度分 = (TTFT归一化 + TPS归一化) / 2
```

## 三、测试方法细节

### API参数
```json
{
  "model": "<模型名>",
  "messages": [{"role": "user", "content": "<测试prompt>"}],
  "stream": true,
  "max_tokens": 1024,
  "temperature": 0.3
}
```

### 超时配置
| 参数 | NVIDIA模型 | GLM模型 |
|------|-----------|---------|
| HTTP连接超时 | 14s | 25s |
| Stream截止时间(R1) | 22s | 40s |
| Stream截止时间(R2/R3) | - | 20s |
| 请求间隔 | 0.5s | 0.2s |

### 重试策略（三轮重试）
1. **Round 1**：全量测试（48项 = 8模型 × 6维度）
2. **Round 2**：重试失败项，但跳过R1失败≥4次的模型（避免无效等待）
3. **Round 3**：最终重试仍失败项
4. **历史回退**：三轮全部失败后，沿用之前评测的历史分数

### 关键优化（v2脚本）
- **GLM模型TTFT普遍12-32秒**，需要更长的stream deadline
- **glm-4.7-flash/flashx超时率极高（10-15次/48项）**，R2/R3直接跳过节省时间
- **历史分数回退**避免网络波动导致0分误判（如glm-5.1数学首次截断得1分，完整回答实际9分）
- 总耗时从~40分钟优化到~20分钟

## 四、评分函数

### 中文写作（score_chinese）
```
字数≥250: +3 | ≥150: +2 | ≥80: +1
含"量子比特": +2 | 含"叠加态": +2 | 含"量子纠缠": +2
含"基本原理"等: +1
```

### 逻辑推理（score_reasoning）
```
含"热/温度/烫": +3（触摸法核心）
含"等/等待": +2（等待策略）
含"第一/第二/第三"等编号: +2
同时含"开"和"关": +2
总分≥7额外: +1
```

### 代码生成（score_coding）
```
含"def ": +1 | 含queue/deque/collections: +2
含"bfs/level/层序": +1
含"append": +1 | 含"pop/popleft": +1
含"result/res/levels/ans": +1
while+queue: +2 | for+range: +1
```

### 数学（score_math）
```
同时含"-2"和"-3": +3（正确解）
含"(x+2)"等格式: +2 | 含"(x+3)"等格式: +2
含"因式分解/分解": +1
含"验证/代入/检验": +1
```

### 指令遵循（score_instruction）
```
恰好5个条目: +3 | ≥4: +2 | ≥3: +1
无编号: +2
字母排序: +3
总行数≤6: +2 | ≤8: +1
```

### 幻觉检测（score_hallucination）
```
含"guido/van rossum/范罗苏姆": +2（Python之父正确）
含"1991/1990": +1（发布年份正确）
含"不存在/无法确认/虚构"等拒绝词: +4（正确拒绝虚构人物）
含"hopfield/hinton/霍普菲尔德/辛顿": +3（2024诺贝尔物理学奖正确）
```

## 六、关键发现

### 1. GLM-5.1 质量第一但速度最慢
- 总分59/60（仅数学9分），质量分全排行第一
- TTFT 16秒，TPS仅15，综合分被速度严重拖累
- 适合作为OpenClaw主模型（质量优先），不适合高并发子agent

### 2. NVIDIA模型速度碾压GLM
- minimax-m2.5 TPS 428.3（GLM最快仅35），TTFT普遍<3秒
- 免费API + 高速 = 子agent首选

### 3. GLM系列数学能力普遍偏弱
- 仅glm-5.1(9分)和glm-4.6v-flash(9分)数学合格
- glm-5-turbo仅1分，其余GLM模型0-2分

### 4. GLM幻觉检测突出
- glm-5.1、glm-5-turbo均满分10分（正确拒绝虚构人物）
- gpt-oss-120b仅0分（编造了虚构人物的信息）

### 5. glm-4.7-flash/flashx不可用于生产
- 超时率>20%，总分仅10-14分
- 已从OpenClaw配置中移除

### 6. glm-4.6v-flash（免费）性价比极高
- 总分50，综合7.0，排名第8
- 免费多模态模型，数学9分
- 适合作为轻量级子agent

## 七、OpenClaw配置建议

### 当前配置
- **主模型**: g/glm-5.1（质量优先）
- **子agent池**: g/glm-5-turbo → g/glm-4.7 → g/glm-4.6v-flash → NVIDIA轮询

- glm-4.7-flash、glm-4.7-flashx、glm-4.6（超时率过高）
